# Omega Topology Publishing Plan

Purpose

Generate topology-aware posts from omega_infinity/api_map.md
and store them in workspace/posts/ using platform-specific frequency.


Source Context

omega_infinity/api_map.md


Output Directory

omega_infinity/posts/


Naming Convention

YYYY-MM-DD_omega_platform.md

Examples:

2026-04-09_omega_bluesky.md
2026-04-09_omega_x.md
2026-04-13_omega_medium.md
2026-04-13_omega_substack.md


Platform Schedule


Bluesky

frequency: daily
style: short topology insight
length: 3–8 lines
goal: conceptual clarity and stack-aware statements


X

frequency: daily
style: compressed topology claim
length: <= 280 characters
goal: anchor sentence or topology constraint observation


Medium

frequency: weekly
day: Wednesday
style: explanatory topology article
length: medium-form structured explanation
goal: evidence-layer reasoning discussion


Substack

frequency: weekly
day: Sunday
style: longform topology essay
length: architecture-level reasoning narrative
goal: Omega Infinity framework development


Generation Rules

1. Always read omega_infinity/api_map.md before writing
2. Preserve topology-aware reasoning structure
3. Keep interpretation downstream from evidence structure
4. Avoid rhetorical certainty beyond stack closure
5. Maintain platform-specific tone differences


Execution Instruction

Generate posts only for platforms scheduled for today
and save outputs using the naming convention inside workspace/posts/.
